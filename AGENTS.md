# Codex 规则

## 提交
用户说“提交”即执行完整发布：`git add -A` → `go test ./...` 通过 → 删除所有 `_test.go` 测试文件 → `go mod tidy` → 更新 `README.md` 保持文档与代码一致 → `git commit`（中文信息，未指定则按变更自动生成）→ `git push`（未指定分支则推当前分支到同名远程）。README 改动与代码改动一并提交。测试失败若因依赖冲突，先修 `go.mod/go.sum` 再重试。

多仓库联动时：先将本仓库引用的自有模块升级到最新可用版本，再从最底层 `go.mod` 仓库开始，逐个「测试 → 提交 → 打 tag」，然后按依赖方向逐级向上（上层升级依赖后再提交打 tag）。

## 打 tag
用户说“打 tag”即执行 `make tag`。

## 注释
新增或修改代码时，注释统一中文：每个方法补中文方法注释，关键逻辑、边界条件、异常分支补中文行内注释。
